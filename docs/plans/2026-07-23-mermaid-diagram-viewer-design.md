# Mermaid 图表查看器设计

## 问题与目标

当前 Mermaid SVG 被限制在正文列宽内。E-R 图包含较多实体时，整张图会按比例缩小；文档大纲展开后正文更窄，文字进一步缩小。目标是在保留正文全图预览的同时，为所有 Mermaid 图表提供不失真的放大、缩小和拖动能力。

## 方案

正文中的 Mermaid 图表继续作为全图预览。渲染完成后，为图表增加可点击和键盘操作入口；点击或按 Enter 后打开独立图表查看器。查看器复用 Mermaid 已生成的 SVG，不重新解析 Markdown，也不引入新的第三方依赖。

查看器首次打开时按可用区域完整显示图表。鼠标滚轮围绕指针位置缩放，拖动调整位置；工具栏提供放大、缩小、适应窗口和关闭按钮，并显示当前缩放比例。双击图表可快速放大，`Esc` 关闭，`+`、`-` 和 `0` 分别对应放大、缩小和适应窗口。

## 边界与验证

缩放下限为适应窗口的比例，上限为原始 SVG 的四倍。拖动时至少保留部分图表在可视区域，防止图表完全丢失。窗口尺寸变化时重新适应窗口；关闭查看器后恢复页面滚动。

实现不修改 Mermaid 源码、不改变 Markdown 文件内容，也不影响普通图片。验证包括缩放计算、类型检查、生产构建、macOS 打包、应用签名和 DMG 完整性检查。受 IOA 限制，最终鼠标和触控板交互由用户手动验收。
