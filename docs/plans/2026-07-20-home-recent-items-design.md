# 首页最近打开记录设计

## 目标

欢迎页在保留“打开文件夹”“在新窗口打开文件夹”“打开文件”入口的同时，展示最近打开的文件和文件夹。用户可以直接从首页继续上次的阅读，不必再经过系统文件选择器。

## 方案

继续复用 `recent-files.json` 中现有的路径列表，不迁移数据，也不增加新的持久化格式。主进程读取有效路径后，将其转换为 `{ path, name, kind }` 结构，其中 `kind` 为 `file` 或 `folder`。渲染进程只能通过 preload 暴露的 IPC API 获取和打开这些记录，不直接访问文件系统。

首页将记录分为 Recent folders 和 Recent files 两组，保持原有的最近优先顺序，每组展示名称和完整路径。点击文件夹或文件时在当前窗口打开；“Open Folder in New Window”仍作为独立按钮保留。菜单清空最近记录、打开新文件或文件夹后，主进程向所有窗口广播最新列表，停留在首页的窗口会立即更新。

## 视觉与状态

整体延续暖白背景、深棕衬线标题和青绿色操作色，但缩小原先占满屏幕的标题，为最近记录面板提供稳定空间。最近记录使用轻量分栏和细边框，避免做成通用仪表盘卡片。窄窗口下改为单列，路径截断并通过 `title` 保留完整信息。

没有记录时仍显示两个分组及简短空状态，帮助用户理解功能。已失效的路径沿用现有逻辑自动过滤，不在首页出现。验证包括 TypeScript 类型检查、生产构建、macOS 打包和 DMG 完整性校验；受 IOA 限制，应用界面由用户手动验收。
